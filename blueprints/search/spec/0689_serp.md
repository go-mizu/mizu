# SERP API Automation (`pkg/serp` + `cli/serp.go`)

## Goal

Automate registration on SerpAPI using temporary mail.tm emails, store API keys locally, and provide a CLI for searching and managing accounts.

## Overview

- **mail.tm** – disposable email provider with a REST API; used to create temp emails for signup
- **SerpAPI** – provides real-time Google Search JSON results; free tier = 100 searches/month/account
- Registration flow: create temp email → sign up on SerpAPI → verify email → extract API key → store locally
- Key rotation: auto-remove keys with 0 searches left; warn when < 10 remain; register `--count N` to bulk-add

## Data Storage

File: `$HOME/data/serp/keys.json`

```json
{
  "accounts": [
    {
      "email": "abc@mail.tm",
      "password": "random-password",
      "api_key": "abc123...",
      "registered_at": "2026-03-08T00:00:00Z",
      "searches_left": 87,
      "last_checked": "2026-03-08T12:00:00Z"
    }
  ]
}
```

## mail.tm API

Base URL: `https://api.mail.tm`

- `GET /domains` → list available domains, pick first
- `POST /accounts` body: `{"address":"user@domain","password":"pass"}` → create account
- `POST /token` body: `{"address":"user@domain","password":"pass"}` → `{"token":"...","id":"..."}`
- `GET /messages` header: `Authorization: Bearer <token>` → list received messages
- `GET /messages/{id}` → get message body (HTML/text with verification link)

## SerpAPI HTTP Registration

SerpAPI uses Rails-based web app. Registration endpoint:

- `POST https://serpapi.com/users` with JSON body:
  ```json
  {"user": {"email": "...", "password": "...", "password_confirmation": "..."}}
  ```
  Headers: `Content-Type: application/json`, `Accept: application/json`

- Response 200: account created, verification email sent
- Response 4xx: likely bot-blocked → fall back to rod browser

Email verification: SerpAPI sends email with link like `https://serpapi.com/users/confirmation?confirmation_token=...`
- `GET <verification_link>` → confirms account

Login to get API key:
- `POST https://serpapi.com/users/sign_in.json` body: `{"user":{"email":"...","password":"..."}}`
- Response includes `api_key` field

Alternative: after verification, `GET https://serpapi.com/account.json?api_key=<key>` to verify key works.

## SerpAPI Search

```
GET https://serpapi.com/search.json?q=<query>&api_key=<key>&engine=google
```

Response: JSON with `organic_results`, `search_metadata`, etc.

## SerpAPI Account Status

```
GET https://serpapi.com/account.json?api_key=<key>
```

Returns: `plan_searches_left`, `total_searches_left`, `this_month_usage`, `api_key`

## CLI Commands

```
search serp register [--count N]     Register N accounts (default 1)
search serp search <query>           Search using random available key
search serp list                     List all stored keys with status
search serp status                   Refresh searches_left from SerpAPI for all keys
```

## Key Selection Logic (search command)

1. Load `keys.json`
2. Remove accounts with `searches_left == 0` (save back)
3. Filter: `searches_left >= 10` as "available"
4. If none available: error "no available keys, run `serp register`"
5. Pick random from available
6. Execute search
7. Decrement `searches_left` by 1, update `last_checked`, save

## Rod Fallback

If HTTP registration fails (non-2xx or blocked response):
- Launch headless rod browser
- Navigate to `https://serpapi.com/users/sign_up`
- Fill form fields: email, password, password_confirmation
- Submit form
- Continue with email verification via mail.tm API

## Error Handling

- mail.tm: retry up to 3x on network errors; poll messages for up to 90s (3s interval)
- SerpAPI HTTP: if 4xx on register, log warning and try rod fallback
- rod fallback: 30s timeout for page navigation
- Verification link extraction: regex `https://serpapi\.com/users/confirmation\?confirmation_token=[^\s"<]+`
