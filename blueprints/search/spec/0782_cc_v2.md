# 0782 вҖ” CC Pipeline v2: Complete Rewrite

## Status: Accepted
## Date: 2026-03-22

---

## 1. Motivation

The v1 CC pipeline (`cli/cc_publish*.go` + `cli/cc_redis.go`) has critical bugs that are
difficult to fix due to tangled state management:

1. **WARC deletion race**: Multiple code paths delete raw `.warc.gz` files вҖ” pipeline cleanup,
   watcher cleanup (Pass 1, Pass 2), prefetch goroutine leaks. A prefetch goroutine can
   `os.Remove()` a file that another pipeline session is about to pack.

2. **Prefetch goroutine leak**: When pack fails and the loop does `continue`,
   `prefetchCancel` is set to nil without calling it. The leaked goroutine runs indefinitely.

3. **Scattered state**: Committed set lives in stats.csv (file), Redis SADD, watcher_status.json,
   and in-memory maps вҖ” 4 sources of truth that can diverge.

4. **Overly complex cleanup**: 3 cleanup passes + sidecar files + legacy glob fallbacks.
   Each pass has race conditions with concurrent pipeline sessions.

5. **Monolithic functions**: `ccRunPipeline` is 250+ lines with inline prefetch, download,
   pack, export, cleanup, and Redis updates. `ccWatcherFlush` is 280+ lines.

6. **No file locking**: Multiple processes can download the same WARC simultaneously,
   or delete a file another process has open.

### Goal

Rewrite as `pkg/cc_v2` вҖ” a clean, testable package with:
- **Single source of truth**: Redis is the authority; file-based fallback is read-only
- **No WARC races**: File locking + atomic state transitions prevent concurrent access
- **Simple lifecycle**: Download вҶ’ Lock вҶ’ Pack вҶ’ Unlock+Delete. No sidecar files.
- **Structured logging**: All events as structured JSON for debugging
- **100+ shards/hour** committed to HuggingFace (same target, simpler path)

### Three Processes (same as v1, cleaner boundaries)

| Process | Responsibility | Stdin/Stdout |
|---------|---------------|--------------|
| **Pipeline** | Download WARC вҶ’ pack вҶ’ parquet. One shard at a time, sequential. | Logs to stdout |
| **Watcher** | Poll parquet dir вҶ’ batch вҶ’ commit to HF. Single instance per server. | Logs to stdout |
| **Scheduler** | Manage pipeline screen sessions. Auto-heal, adaptive scaling. | Logs to stdout + file |

### Key Design Decisions

1. **No prefetch**: v1's prefetch added 15% throughput but introduced goroutine leaks and
   race conditions. v2 processes shards sequentially. If throughput is insufficient, we can
   add prefetch later with proper cancellation вҖ” but only after measuring.

2. **Atomic parquet writes**: Write to `.parquet.tmp` then `os.Rename()`. Watcher only sees
   complete files (same as v1, but enforced at the type level).

3. **File locking via Redis**: Before downloading a WARC, acquire a Redis lock
   (`SETNX cc:v2:{crawl}:lock:{idx}`). This prevents two pipeline sessions from downloading
   the same shard. Lock has a 30-minute TTL (auto-release on crash).

4. **Delete-after-commit only**: Raw WARCs are ONLY deleted by the watcher AFTER the parquet
   is committed to HF. Pipeline never deletes WARCs. No cleanup passes needed.

5. **Single source of truth**: Redis `cc:v2:{crawl}:shards` hash stores per-shard state.
   File-based stats.csv is written by watcher for HF upload but never read for decisions.

## 3. Data Model

### 3.1 Shard States (Redis)

Each shard transitions through a simple state machine:

```
         в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
         в”Ӯ  (none)  в”Ӯ  вҶҗ not in Redis = not started
         в””в”Җв”Җв”Җв”Җв”¬в”Җв”Җв”Җв”Җв”Җв”ҳ
              в”Ӯ pipeline claims
         в”Ңв”Җв”Җв”Җв”Җв–јв”Җв”Җв”Җв”Җв”Җв”җ
         в”Ӯ claimed  в”Ӯ  вҶҗ SETNX lock, pipeline downloading/packing
         в””в”Җв”Җв”Җв”Җв”¬в”Җв”Җв”Җв”Җв”Җв”ҳ
              в”Ӯ parquet written
         в”Ңв”Җв”Җв”Җв”Җв–јв”Җв”Җв”Җв”Җв”Җв”җ
         в”Ӯ  ready   в”Ӯ  вҶҗ parquet exists, waiting for watcher
         в””в”Җв”Җв”Җв”Җв”¬в”Җв”Җв”Җв”Җв”Җв”ҳ
              в”Ӯ watcher commits to HF
         в”Ңв”Җв”Җв”Җв”Җв–јв”Җв”Җв”Җв”Җв”Җв”Җв”җ
         в”Ӯ committed в”Ӯ  вҶҗ on HF, local files can be deleted
         в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

On failure at any stage, the lock expires (30-min TTL) and the shard returns to `(none)`.

## 4. Pipeline Worker

### 4.1 Core Loop

```go
func (p *Pipeline) Run(ctx context.Context, indices []int) error {
    for _, idx := range indices {
        if ctx.Err() != nil {
            return ctx.Err()
        }

        // 1. Skip if already committed or ready
        if p.store.IsCommitted(ctx, idx) || p.store.IsReady(ctx, idx) {
            p.log.Info("skip", "shard", idx, "reason", "already done")
            continue
        }

        // 2. Claim the shard (distributed lock)
        if !p.store.Claim(ctx, idx) {
            p.log.Info("skip", "shard", idx, "reason", "claimed by another")
            continue
        }

        // 3. Download WARC
        warcPath, dur, err := p.download(ctx, idx)
        if err != nil {
            p.store.Release(ctx, idx) // release lock
            if p.skipErrors {
                p.log.Warn("download failed, skipping", "shard", idx, "err", err)
                continue
            }
            return fmt.Errorf("shard %d download: %w", idx, err)
        }

        // 4. Pack to parquet (atomic write via .tmp)
        pqPath, stats, err := p.pack(ctx, idx, warcPath)
        if err != nil {
            p.store.Release(ctx, idx)
            if p.skipErrors {
                p.log.Warn("pack failed, skipping", "shard", idx, "err", err)
                continue
            }
            return fmt.Errorf("shard %d pack: %w", idx, err)
        }

        // 5. Mark ready (watcher will pick it up)
        p.store.MarkReady(ctx, idx, pqPath, warcPath, stats)
        p.log.Info("ready", "shard", idx, "rows", stats.Rows, "size", stats.PqBytes)
    }
    return nil
}
```

### 4.2 Download

```go
func (p *Pipeline) download(ctx context.Context, idx int) (string, time.Duration, error) {
    t0 := time.Now()

    // Check if WARC already on disk (from a previous crashed run)
    if path := p.findWARC(idx); path != "" {
        return path, 0, nil
    }

    // Resolve manifest path
    remotePath, err := p.manifest.Path(idx)
    if err != nil {
        return "", 0, err
    }

    // Download to warc/ directory
    localPath := filepath.Join(p.warcDir, filepath.Base(remotePath))
    tmpPath := localPath + ".tmp"

    if err := p.ccClient.DownloadFile(ctx, remotePath, tmpPath, nil); err != nil {
        os.Remove(tmpPath) // cleanup partial
        return "", 0, err
    }

    // Atomic rename
    if err := os.Rename(tmpPath, localPath); err != nil {
        return "", 0, err
    }

    p.store.RecordEvent(ctx, "downloaded")
    return localPath, time.Since(t0), nil
}
```

Key improvements over v1:
- Download to `.tmp` then rename (atomic)
- No sidecar files вҖ” path stored in Redis on `MarkReady`
- Single download location, no fallback glob patterns

### 4.3 Pack

Uses existing `warcmd.PackConfig` + `packDirectToParquet()` вҖ” no changes needed
to the actual conversion logic. Just wraps it with atomic file writes.

```go
func (p *Pipeline) pack(ctx context.Context, idx int, warcPath string) (string, *ShardStats, error) {
    t0 := time.Now()
    shard := fmt.Sprintf("%05d", idx)
    pqPath := filepath.Join(p.parquetDir, shard+".parquet")
    tmpPath := pqPath + ".tmp"

    // Remove stale tmp from previous crash
    os.Remove(tmpPath)

    cfg := warcmd.PackConfig{
        InputFiles:   []string{warcPath},
        Workers:      0,
        Force:        true,
        LightConvert: true,
        StatusCode:   200,
        MIMEFilter:   "text/html",
        MaxBodySize:  512 * 1024,
    }

    rows, htmlBytes, mdBytes, packStats, err := packDirectToParquet(ctx, cfg, tmpPath, p.progressFn(shard))
    if err != nil {
        os.Remove(tmpPath)
        return "", nil, err
    }

    // Atomic rename
    if err := os.Rename(tmpPath, pqPath); err != nil {
        return "", nil, err
    }

    fi, _ := os.Stat(pqPath)
    pqBytes := int64(0)
    if fi != nil {
        pqBytes = fi.Size()
    }

    stats := &ShardStats{
        Rows:      rows,
        HTMLBytes:  htmlBytes,
        MDBytes:    mdBytes,
        PqBytes:   pqBytes,
        DurDlS:    0, // set by caller
        DurPackS:  int64(time.Since(t0).Seconds()),
        PeakRSSMB: int64(warcmd.ReadRSSMB()),
    }

    // Write .meta.json sidecar for watcher (backup if Redis is down)
    metaPath := filepath.Join(p.parquetDir, shard+".meta.json")
    metaJSON, _ := json.Marshal(stats)
    os.WriteFile(metaPath, metaJSON, 0o644)

    p.store.RecordEvent(ctx, "packed")
    return pqPath, stats, nil
}
```

## 5. Watcher

### 5.1 Core Loop

```go
func (w *Watcher) Run(ctx context.Context) error {
    // Startup: sync committed set, purge stale files
    w.syncFromHF(ctx)

    ticker := time.NewTicker(w.pollInterval)
    defer ticker.Stop()

    for {
        select {
        case <-ctx.Done():
            return ctx.Err()
        case <-ticker.C:
            if err := w.flush(ctx); err != nil {
                w.log.Error("flush", "err", err)
            }
        }
    }
}
```

### 5.2 Flush Cycle

```go
func (w *Watcher) flush(ctx context.Context) error {
    // 1. Scan parquet dir for new .parquet files (not .parquet.tmp)
    files := w.scanParquetDir()

    // 2. Filter: skip committed, skip too small (<100 bytes)
    var pending []ParquetFile
    for _, f := range files {
        if w.store.IsCommitted(ctx, f.Idx) {
            os.Remove(f.Path) // cleanup leftover
            continue
        }
        if f.Size < 100 {
            w.log.Warn("corrupt parquet, deleting", "shard", f.Idx, "size", f.Size)
            os.Remove(f.Path)
            continue
        }
        pending = append(pending, f)
    }

    if len(pending) == 0 {
        return nil
    }

    // 3. Rate limit check
    if !w.rateLimiter.Allow() {
        w.log.Info("rate limited, holding", "pending", len(pending))
        return nil
    }

    // 4. Cap batch size
    if len(pending) > w.maxBatch {
        pending = pending[:w.maxBatch]
    }

    // 5. Write stats, merge from HF, build commit ops
    w.writeStats(pending)
    w.mergeFromHF(ctx)
    w.regenerateREADME()

    // 6. Commit to HF with retry
    commitURL, err := w.commitWithRetry(ctx, pending)
    if err != nil {
        return err
    }

    // 7. On success: mark committed, delete local files
    for _, f := range pending {
        w.store.MarkCommitted(ctx, f.Idx)

        // Delete parquet + meta
        os.Remove(f.Path)
        os.Remove(f.MetaPath)

        // Delete raw WARC (path from Redis)
        if warcPath := w.store.GetWARCPath(ctx, f.Idx); warcPath != "" {
            os.Remove(warcPath)
        }
    }

    w.log.Info("committed", "url", commitURL, "shards", len(pending))
    return nil
}
```

Key improvements over v1:
- **WARC deletion only here**: Pipeline never deletes WARCs. Only watcher deletes after HF commit.
- **WARC path from Redis**: No sidecar files, no glob patterns, no legacy fallbacks.
- **No cleanup passes**: No Pass 1/2/3. Just delete what was committed in this batch.
- **Simpler rate limiting**: `time.Ticker`-based, not manual timestamp tracking.

## 7. State Store Interface

```go
// Store is the state backend for cc_v2. Redis is primary; file-based is fallback.
type Store interface {
    // Shard lifecycle
    IsCommitted(ctx context.Context, idx int) bool
    IsReady(ctx context.Context, idx int) bool
    Claim(ctx context.Context, idx int) bool       // SETNX with 30-min TTL
    Release(ctx context.Context, idx int)           // DEL lock
    MarkReady(ctx context.Context, idx int, pqPath, warcPath string, stats *ShardStats)
    MarkCommitted(ctx context.Context, idx int)

    // Queries
    CommittedCount(ctx context.Context) int
    CommittedSet(ctx context.Context) map[int]bool
    GetWARCPath(ctx context.Context, idx int) string

    // Telemetry
    RecordEvent(ctx context.Context, event string) // downloaded|packed|committed
    EventRate(ctx context.Context, event string, window time.Duration) float64

    // Watcher status
    SetWatcherStatus(ctx context.Context, status WatcherStatus)
    GetWatcherStatus(ctx context.Context) (WatcherStatus, bool)

    // Logging
    Log(ctx context.Context, source, level, msg string, fields ...any)

    // Lifecycle
    Close() error
}
```

### 7.1 Redis Implementation

```go
type RedisStore struct {
    rdb    *redis.Client
    crawl  string
    host   string // hostname:pid for lock ownership
}

func (s *RedisStore) Claim(ctx context.Context, idx int) bool {
    key := s.key("lock", strconv.Itoa(idx))
    ok, err := s.rdb.SetNX(ctx, key, s.host, 30*time.Minute).Result()
    if err != nil || !ok {
        return false
    }
    // Also set shard state
    s.rdb.HSet(ctx, s.key("shard", strconv.Itoa(idx)), map[string]interface{}{
        "state":      "claimed",
        "claimed_by": s.host,
        "claimed_at": time.Now().UTC().Format(time.RFC3339),
    })
    return true
}

func (s *RedisStore) MarkReady(ctx context.Context, idx int, pqPath, warcPath string, stats *ShardStats) {
    pipe := s.rdb.Pipeline()
    pipe.HSet(ctx, s.key("shard", strconv.Itoa(idx)), map[string]interface{}{
        "state":       "ready",
        "ready_at":    time.Now().UTC().Format(time.RFC3339),
        "pq_path":     pqPath,
        "warc_path":   warcPath,
        "rows":        stats.Rows,
        "html_bytes":  stats.HTMLBytes,
        "md_bytes":    stats.MDBytes,
        "pq_bytes":    stats.PqBytes,
        "dur_dl_s":    stats.DurDlS,
        "dur_pack_s":  stats.DurPackS,
        "peak_rss_mb": stats.PeakRSSMB,
    })
    // Release the lock (pipeline is done with this shard)
    pipe.Del(ctx, s.key("lock", strconv.Itoa(idx)))
    pipe.Exec(ctx)
}

func (s *RedisStore) MarkCommitted(ctx context.Context, idx int) {
    pipe := s.rdb.Pipeline()
    pipe.HSet(ctx, s.key("shard", strconv.Itoa(idx)),
        "state", "committed",
        "committed_at", time.Now().UTC().Format(time.RFC3339))
    pipe.SAdd(ctx, s.key("committed"), idx)
    pipe.Exec(ctx)
}
```

### 7.2 File-Based Fallback

When Redis is unavailable, use file-based locking:

```go
type FileStore struct {
    dataDir string
    crawl   string
}

func (s *FileStore) Claim(ctx context.Context, idx int) bool {
    lockPath := filepath.Join(s.dataDir, "locks", fmt.Sprintf("%05d.lock", idx))
    f, err := os.OpenFile(lockPath, os.O_CREATE|os.O_EXCL|os.O_WRONLY, 0o644)
    if err != nil {
        // Check if lock is stale (>30 min old)
        if fi, serr := os.Stat(lockPath); serr == nil && time.Since(fi.ModTime()) > 30*time.Minute {
            os.Remove(lockPath)
            return s.Claim(ctx, idx) // retry once
        }
        return false
    }
    fmt.Fprintf(f, "%s:%d\n", hostname(), os.Getpid())
    f.Close()
    return true
}
```

### Migration steps:
1. Deploy v2 binary
2. Start v2 watcher + scheduler on fresh shard range

## 14. Risk Analysis

| Risk | Mitigation |
|------|------------|
| Redis down | FileStore fallback with file-based locks |
| Lock leak (process killed) | 30-min TTL auto-expires |
| Watcher crash mid-commit | HF commit is atomic; next run re-scans |
| Two watchers running | Only one should run; add Redis `SETNX` for watcher lock |
| Disk full | Scheduler monitors disk, pauses at < 20 GB |
| HF rate limit | Same retry logic as v1 (429 вҶ’ sleep Retry-After + 30s) |
| No prefetch = slower | Measure first; add if needed with proper context cancel |

4. **Clean shutdown** вҖ” no orphaned goroutines, no leaked locks
