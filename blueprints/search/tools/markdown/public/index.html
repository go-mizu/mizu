How it works

Every URL goes through three tiers, falling back automatically until clean Markdown is produced.

01

Native

Sites that support Accept: text/markdown return it directly. Zero conversion overhead.

02

AI extraction

HTML pages are fetched and converted by an AI model. Pulls out what matters and drops the noise.

03

Browser rendering

React, Vue, Angular, any SPA. Rendered in a real browser first so all content is present before conversion.

Results are cached for 1 hour. CORS is open on all endpoints so you can call from any browser or agent.

FAQ

Why Markdown instead of HTML?

HTML includes navigation, ads, scripts, and styling your agent doesn't need. Markdown strips all of that down to headings, paragraphs, and links — about 80% fewer tokens to process.

Is it really free?

Yes. Open access, no account, no API key. Just use the URL directly in your code or agent prompt.

How do you handle JavaScript-heavy pages?

Pages that require JavaScript fall back to a headless browser that runs the page fully before converting. This handles React, Vue, Angular, and any SPA.

Can I call this from the browser?

Yes. All endpoints include Access-Control-Allow-Origin: * so you can call directly from browser JavaScript.

What about login-protected pages?

Only publicly available URLs are accessible. Pages behind authentication, paywalls, or rate limits will return an error.

What are the limits?

Responses are capped at 5 MB. Requests time out after 10 seconds (30 for browser rendering). Reasonable use is not rate-limited.
