Pipeline

Three tiers. Automatic fallback.

Every URL goes through up to three conversion tiers. Each falls back to the next automatically until clean Markdown is produced.

Native

Requests content via Accept: text/markdown header. Sites that support it return Markdown directly with zero conversion overhead.

AI extraction

HTML is fetched and converted by an AI model. Structure-aware extraction that pulls headings, text, and links while dropping navigation, ads, and scripts.

Browser render

Full headless browser rendering for JavaScript-heavy pages. React, Vue, Angular, any SPA. Content is fully loaded before conversion starts.

Quick start

Works from any language or shell.

Append any URL to the endpoint. Or POST for JSON with metadata including token count, conversion method, and page title.

# get markdown from any URL
$ curl https://markdown.go-mizu.workers.dev/https://example.com

# or POST for JSON with metadata
$ curl -X POST https://markdown.go-mizu.workers.dev/convert \
    -H 'Content-Type: application/json' \
    -d '{"url":"https://example.com"}'

FAQ

Common questions

Why Markdown instead of HTML?

HTML includes navigation, ads, scripts, and styling your agent doesn't need. Markdown keeps headings, paragraphs, and links. About 80% fewer tokens to process.

Is it really free?

Yes. Open access. Use the URL directly in your code or agent prompt. No account, no API key.

How are JS-heavy pages handled?

Pages requiring JavaScript fall back to a headless browser that fully renders the page before converting. Handles React, Vue, Angular, and any SPA.

Can I call from the browser?

Yes. All endpoints return Access-Control-Allow-Origin: *. Call directly from client-side JavaScript.

What about protected pages?

Only publicly available URLs are accessible. Pages behind authentication or paywalls return an error.

What are the limits?

5 MB max response. 10s timeout (30s for browser rendering). Reasonable use is not rate-limited.
