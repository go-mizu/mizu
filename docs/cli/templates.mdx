# Templates

The Mizu CLI includes built-in templates for scaffolding new projects. Each template provides a complete, runnable starting point with idiomatic Go project structure.

## Available Templates

| Template | Description | Use Case |
|----------|-------------|----------|
| [`minimal`](/cli/template-minimal) | Smallest runnable Mizu project | Quick prototypes, learning |
| [`api`](/cli/template-api) | JSON API service with recommended layout | Production APIs, microservices |
| [`contract`](/cli/template-contract) | Transport-neutral service with REST, JSON-RPC, OpenAPI | Multi-protocol services, clean architecture |

## List Templates

```bash
mizu new --list
```

Output:
```
Template   Purpose
api        JSON API service with a recommended layout
contract   Transport-neutral service with REST, JSON-RPC, and OpenAPI
minimal    Smallest runnable Mizu project
```

## Using Templates

### Basic Usage

```bash
# Create project with template
mizu new myapp --template api
```

### With Custom Module Path

```bash
mizu new myapp --template api --module github.com/myorg/myapp
```

### Preview Before Creating

```bash
mizu new myapp --template api --dry-run
```

## Template Variables

Templates use Go's `text/template` syntax. The following variables are available:

| Variable | Description | Example |
|----------|-------------|---------|
| `{{.Name}}` | Project name | `myapp` |
| `{{.Module}}` | Go module path | `github.com/myorg/myapp` |
| `{{.License}}` | License identifier | `MIT` |
| `{{.Year}}` | Current year | `2024` |
| `{{.Vars.key}}` | Custom variables | Any value |

### Setting Variables

```bash
# Set project name (default: derived from path)
mizu new myapp --template api --name my-service

# Set module path (default: example.com/<name>)
mizu new myapp --template api --module github.com/myorg/myapp

# Set license (default: MIT)
mizu new myapp --template api --license Apache-2.0

# Set custom variables
mizu new myapp --template api --var author="John Doe"
```


### `README.md`

Each template generates a README with:
- Project description
- Setup instructions
- Running instructions
- Project structure overview

## Template Architecture

Templates are embedded in the CLI binary using Go's `embed` directive. This means:

- No network required to create projects
- Templates always match CLI version
- Consistent across all platforms

## File Naming

Special file name mappings handle dotfiles:

| Template Name | Output Name |
|---------------|-------------|
| `gitignore` | `.gitignore` |
| `dockerignore` | `.dockerignore` |
| `env` | `.env` |
| `env.example` | `.env.example` |

Files with `.tmpl` extension are processed as templates, and the extension is removed in output.

### Decision Guide

| Scenario | Recommended Template |
|----------|---------------------|
| Learning Mizu | `minimal` |
| Quick prototype | `minimal` |
| Production REST API | `api` |
| Microservice | `api` |
| Custom structure | `minimal` |
| Team project | `api` |
| Multi-protocol API (REST + RPC) | `contract` |
| Transport-neutral services | `contract` |
| Auto-generated OpenAPI | `contract` |
| Clean architecture | `contract` |
