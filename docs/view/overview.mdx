The `view` package provides server-side HTML rendering using Go's built-in `html/template` engine. It adds a thin layer on top for layout support, custom functions, and development-friendly features like hot reloading.

## What is Server-Side Rendering?

When a user visits your website, the server generates the complete HTML page and sends it to their browser. This is called **server-side rendering (SSR)**.

```
User Request       Server              Browser
    │                │                    │
    │  GET /users    │                    │
    │───────────────▶│                    │
    │                │                    │
    │                │ 1. Fetch data      │
    │                │ 2. Render template │
    │                │ 3. Return HTML     │
    │                │                    │
    │   <html>...    │                    │
    │◀───────────────│                    │
    │                │                    │
    │                │    Display page    │
    │                │───────────────────▶│
```

**Benefits of SSR:**
- Fast initial page load (browser displays content immediately)
- SEO-friendly (search engines can read your content)
- Works without JavaScript
- Simple mental model (one language, one place)

## What the View Package Provides

| Feature | Description |
|---------|-------------|
| Template Engine | Wraps Go's `html/template` with convenience methods |
| Layouts | Wrap content in a consistent page structure |
| Custom Functions | Add your own template helpers |
| Hot Reload | Templates reload automatically during development |
| Embedded Files | Bundle templates in your binary for production |

## How It Works

The view package follows a simple flow:

```
┌─────────────────────────────────────────────────────────────┐
│                        Engine                                │
│                                                              │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐  │
│  │   Config     │───▶│   Templates  │───▶│   Output     │  │
│  │              │    │              │    │              │  │
│  │ - Dir        │    │ - home.html  │    │ <html>...    │  │
│  │ - Extension  │    │ - user.html  │    │              │  │
│  │ - Funcs      │    │ - layout.html│    │              │  │
│  │ - Layout     │    │              │    │              │  │
│  └──────────────┘    └──────────────┘    └──────────────┘  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

1. **Configure** - Tell the engine where templates are and how to process them
2. **Load** - Engine parses and caches your template files
3. **Render** - Pass data to a template, get HTML back

## Quick Example

Here's a minimal example showing the view package in action:

**Directory structure:**
```
myapp/
├── main.go
└── views/
    ├── layout.html
    └── home.html
```

**views/layout.html:**
```html
<!DOCTYPE html>
<html>
<head>
    <title>{{.Title}}</title>
</head>
<body>
    {{template "content" .}}
</body>
</html>
```

**views/home.html:**
```html
{{define "content"}}
<h1>Welcome, {{.Name}}!</h1>
<p>You have {{.Count}} messages.</p>
{{end}}
```

**main.go:**
```go
package main

import (
    "github.com/go-mizu/mizu"
    "github.com/go-mizu/mizu/view"
)

func main() {
    // Create the template engine
    engine := view.New(view.Config{
        Dir:           "./views",
        Extension:     ".html",
        DefaultLayout: "layout",
    })

    // Create the app
    app := mizu.New()

    // Add engine to app (makes it available in handlers)
    engine.Middleware(app)

    // Define a route
    app.Get("/", func(c *mizu.Ctx) error {
        return view.From(c).Render(c, "home", view.Data{
            "Title": "My App",
            "Name":  "Alice",
            "Count": 5,
        })
    })

    app.Listen(":8080")
}
```

When you visit `http://localhost:8080`, you'll see:
```html
<!DOCTYPE html>
<html>
<head>
    <title>My App</title>
</head>
<body>
    <h1>Welcome, Alice!</h1>
    <p>You have 5 messages.</p>
</body>
</html>
```

## Core Concepts

### Engine

The `Engine` is the main object. You create one, configure it, and use it to render templates.

```go
engine := view.New(view.Config{
    Dir:       "./views",
    Extension: ".html",
})
```

### Templates

Templates are HTML files with special syntax for inserting dynamic content. They use Go's `html/template` package.

```html
<h1>Hello, {{.Name}}!</h1>
```

The `{{.Name}}` part is replaced with actual data when rendering.

### Layouts

A layout is a template that wraps around your content. It typically contains the `<html>`, `<head>`, and `<body>` tags that every page needs.

```html
<!-- layout.html -->
<!DOCTYPE html>
<html>
<body>
    {{template "content" .}}  <!-- Your content goes here -->
</body>
</html>
```

### Data

Data is a `map[string]any` that you pass to templates. Templates can access this data using the dot notation.

```go
view.Data{
    "Title":   "Home",
    "User":    currentUser,
    "Items":   []string{"a", "b", "c"},
}
```

## When to Use View

**Use the view package when:**
- Building traditional server-rendered websites
- Creating admin dashboards
- Building email templates
- Generating any HTML from Go

**Consider alternatives when:**
- Building single-page applications (SPAs) with React/Vue/etc.
- Building APIs that return JSON
- Using a separate frontend framework
