# SDK Generation Overview

Mizu Contract includes built-in SDK generators that create native, fully-typed client libraries directly from your Go service definitions. Unlike OpenAPI-based generators, these produce idiomatic code tailored for each language with an OpenAI-style developer experience.

## What is SDK Generation?

SDK generation is the process of automatically creating client libraries from your API definition. Instead of writing HTTP client code by hand for each language, you define your API once in Go and generate clients automatically.

```
Your Go Service (contract.Service)
           │
           ▼
    ┌──────────────────┐
    │  SDK Generator   │
    │   (Go/Py/TS)     │
    └──────────────────┘
           │
           ▼
  Native Client Library
  (Types + Client + Streaming)
```

## Why Use Native SDK Generators?

### Benefits Over Manual Client Code

| Aspect | Manual Code | Generated SDK |
|--------|-------------|---------------|
| **Type Safety** | Error-prone, manual sync | Always matches server |
| **Maintenance** | Update everywhere | Regenerate once |
| **Consistency** | Varies by author | Uniform patterns |
| **Time to Ship** | Hours per language | Seconds |

### Benefits Over OpenAPI-Based Generation

OpenAPI generators are powerful but generic. Mizu's native generators produce:

| Feature | OpenAPI Generators | Mizu SDK Generators |
|---------|-------------------|---------------------|
| **Code Style** | Generic, verbose | Idiomatic, minimal |
| **Dependencies** | Often many | Zero (stdlib only) |
| **API Feel** | HTTP-centric | Resource-oriented |
| **Streaming** | Limited support | First-class SSE |
| **Size** | Large output | Compact, focused |

<Info>
**When to use OpenAPI?** OpenAPI generators support 40+ languages and have mature tooling. Use them when you need languages Mizu doesn't support natively, or when you need OpenAPI compatibility with other tools. See [Client Generation](/contract/client-generation) for OpenAPI approaches.
</Info>

## Quick Comparison

| Feature | Go SDK | Python SDK | TypeScript SDK |
|---------|--------|------------|----------------|
| **Package** | `sdkgo` | `sdkpy` | `sdkts` |
| **Output** | Single `.go` file | uv project | npm package |
| **HTTP Client** | `net/http` | `httpx` | native `fetch` |
| **Streaming** | `EventStream[T]` | Iterator/AsyncIterator | `AsyncIterable<T>` |
| **Dependencies** | None (stdlib) | httpx | None |
| **Runtimes** | Go 1.18+ | Python 3.8+ | Node/Bun/Deno |

## How It Works

### Step 1: Define Your Service

First, define your API contract in Go:

```go
package todo

import "time"

// API defines the Todo service interface
type API interface {
    Create(input CreateInput) (Todo, error)
    List() (ListOutput, error)
    Get(id string) (Todo, error)
    Delete(id string) error
}

type Todo struct {
    ID        string    `json:"id"`
    Title     string    `json:"title"`
    Done      bool      `json:"done"`
    CreatedAt time.Time `json:"created_at"`
}

type CreateInput struct {
    Title string `json:"title"`
}

type ListOutput struct {
    Items []Todo `json:"items"`
    Total int    `json:"total"`
}
```

### Step 2: Register with Contract

Register your implementation to create a service descriptor:

```go
package main

import (
    "yourapp/todo"
    contract "github.com/go-mizu/mizu/contract/v2"
)

func main() {
    impl := todo.NewService() // Your implementation
    svc := contract.Register[todo.API](impl,
        contract.WithDefaultResource("todos"),
    )

    // svc is now a *contract.Service that SDK generators can use
}
```

### Step 3: Generate SDKs

Use the SDK generators to create client libraries:

<CodeGroup>
```go Go SDK
import sdkgo "github.com/go-mizu/mizu/contract/v2/sdk/go"

files, err := sdkgo.Generate(svc, &sdkgo.Config{
    Package: "todoclient",
})
// files[0].Content contains the generated Go code
```

```go Python SDK
import sdkpy "github.com/go-mizu/mizu/contract/v2/sdk/py"

files, err := sdkpy.Generate(svc, &sdkpy.Config{
    Package: "todoclient",
    Version: "1.0.0",
})
// files contains pyproject.toml, _client.py, _types.py, etc.
```

```go TypeScript SDK
import sdkts "github.com/go-mizu/mizu/contract/v2/sdk/ts"

files, err := sdkts.Generate(svc, &sdkts.Config{
    Package: "todoclient",
    Version: "1.0.0",
})
// files contains package.json, src/index.ts, src/_client.ts, etc.
```
</CodeGroup>

### Step 4: Use the Generated Client

<CodeGroup>
```go Go
client := todoclient.NewClient("http://localhost:8080")

todo, err := client.Todos.Create(ctx, &todoclient.CreateInput{
    Title: "Learn Mizu SDKs",
})
```

```python Python
from todoclient import OpenAI

client = OpenAI(base_url="http://localhost:8080")

todo = client.todos.create(title="Learn Mizu SDKs")
print(f"Created: {todo.id}")
```

```typescript TypeScript
import { TodoService } from 'todoclient';

const client = new TodoService({ baseURL: 'http://localhost:8080' });

const todo = await client.todos.create({ title: 'Learn Mizu SDKs' });
console.log(`Created: ${todo.id}`);
```
</CodeGroup>

## Choosing the Right Approach

Use this decision matrix to choose between SDK generation and OpenAPI:

| If you need... | Use |
|----------------|-----|
| Go, Python, or TypeScript client | **Native SDK generators** |
| Maximum type safety | **Native SDK generators** |
| Minimal dependencies | **Native SDK generators** |
| Streaming (SSE) support | **Native SDK generators** |
| Other languages (Java, Ruby, etc.) | OpenAPI generators |
| OpenAPI ecosystem compatibility | OpenAPI generators |
| Both approaches | Both! Generate native SDKs and serve `/openapi.json` |

## Common Patterns

### Build-Time Generation

Generate SDKs as part of your build:

```makefile
# Makefile
.PHONY: sdk
sdk:
	go run ./cmd/gensdks
```

```go
// cmd/gensdks/main.go
package main

import (
    "os"
    "path/filepath"

    "yourapp/todo"
    contract "github.com/go-mizu/mizu/contract/v2"
    sdkgo "github.com/go-mizu/mizu/contract/v2/sdk/go"
    sdkpy "github.com/go-mizu/mizu/contract/v2/sdk/py"
    sdkts "github.com/go-mizu/mizu/contract/v2/sdk/ts"
)

func main() {
    impl := todo.NewService()
    svc := contract.Register[todo.API](impl)

    // Generate Go SDK
    goFiles, _ := sdkgo.Generate(svc, &sdkgo.Config{Package: "todoclient"})
    os.WriteFile("sdk/go/client.go", []byte(goFiles[0].Content), 0644)

    // Generate Python SDK
    pyFiles, _ := sdkpy.Generate(svc, &sdkpy.Config{Package: "todoclient"})
    for _, f := range pyFiles {
        path := filepath.Join("sdk/python", f.Path)
        os.MkdirAll(filepath.Dir(path), 0755)
        os.WriteFile(path, []byte(f.Content), 0644)
    }

    // Generate TypeScript SDK
    tsFiles, _ := sdkts.Generate(svc, &sdkts.Config{Package: "todoclient"})
    for _, f := range tsFiles {
        path := filepath.Join("sdk/typescript", f.Path)
        os.MkdirAll(filepath.Dir(path), 0755)
        os.WriteFile(path, []byte(f.Content), 0644)
    }
}
```

### CI/CD Integration

Regenerate SDKs on API changes:

```yaml
# .github/workflows/sdk.yml
name: Generate SDKs

on:
  push:
    paths:
      - 'api/**'
      - 'internal/todo/**'

jobs:
  generate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-go@v4
        with:
          go-version: '1.22'

      - name: Generate SDKs
        run: make sdk

      - name: Commit SDKs
        run: |
          git add sdk/
          git commit -m "chore: regenerate SDKs" || true
          git push
```
