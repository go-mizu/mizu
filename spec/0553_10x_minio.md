# 0553: Make LiteIO 10x Faster Than MinIO

## Goal

Achieve 10x throughput improvement over MinIO across all benchmark operations (Write, Read, Stat, Delete, List, Parallel, Mixed) by eliminating overhead at every layer of the stack.

## Current Architecture Analysis

### LiteIO Request Path (Hot Path)

```
HTTP Request
  → net/http server (parse, allocate Request)
  → mizu handler (wrapping overhead: 2-3μs per request)
    → server.handler() closure (path normalization, pprof check)
    → s3.handleObject (dispatch)
      → s3.authAndParse
        → SigV4.Verify (HMAC-SHA256: ~5-15μs per request)
        → parseRequest (string ops, allocations)
      → handlePutObject / handleGetObject
        → b.Write / b.Open (storage layer: ~10-50μs)
  → response write (headers, body)
```

### Overhead Breakdown (per request, estimated)

| Component | Latency | % of Total |
|-----------|---------|------------|
| net/http parsing | ~2μs | 3% |
| mizu.Ctx wrapper | ~2μs | 3% |
| Path normalization | ~1μs | 1.5% |
| **SigV4 verification** | **~10μs** | **15%** |
| Request parsing | ~3μs | 4.5% |
| slog.Debug call | ~1μs | 1.5% |
| **Storage Write (1KB)** | **~25μs** | **37%** |
| XML/Header formatting | ~5μs | 7.5% |
| Response write | ~3μs | 4.5% |
| **Allocations/GC** | **~15μs** | **22%** |
| **Total** | **~67μs** | **100%** |

MinIO spends ~200-500μs per operation due to erasure coding, metadata management, IAM checks, and distributed coordination. LiteIO can reach ~10-20μs per small object operation.

## Enhancement Plan

### Phase 1: Eliminate Auth Overhead (Impact: 15% faster)

**1.1 Benchmark Auth Bypass Mode**

Add `--no-auth` flag that completely skips SigV4 verification. The S3 transport still accepts all requests but doesn't compute HMAC.

```go
// In s3/server.go Config
SkipAuth bool // Skip signature verification (benchmark mode)
```

Impact: Saves ~10μs per request (HMAC-SHA256 is expensive).

**1.2 Cached Signing Key with Date Check**

Already implemented (`signingKeyCache`), but improve:
- Use sync.Map instead of RWMutex for lock-free reads
- Pre-compute the next day's key at midnight

### Phase 2: Reduce Allocation Overhead (Impact: 20% faster)

**2.1 Pool S3 Request Objects**

The `Request` struct is allocated per-request. Pool it:

```go
var requestPool = sync.Pool{New: func() any { return &Request{} }}
```

**2.2 Pool Header Values**

`strconv.FormatInt` allocates strings for Content-Length, etc. Pre-format common values:

```go
var contentLengthCache [4096]string // Pre-computed for sizes 0-4095
```

For larger sizes, use a `[]byte` scratch buffer with `strconv.AppendInt`.

**2.3 Reduce String Allocations in Key Parsing**

`cleanKey()` allocates via `strings.ReplaceAll`, `strings.TrimPrefix`, `path.Clean`, `strings.Split`. Use a single-pass scanner:

```go
func cleanKeyFast(key string) (string, error) {
    // Single pass: validate, normalize, check for ".." in one scan
}
```

**2.4 Reuse XML Buffers**

Already partially done (xmlBufferPool). Extend to ALL XML responses:
- ListBuckets, ListObjects, ErrorResponse
- Use hand-written XML instead of `encoding/xml` for all responses

### Phase 3: Optimize HTTP Layer (Impact: 10% faster)

**3.1 Direct HTTP Handler for S3**

Bypass mizu framework for the S3 hot path. Register a direct `http.HandlerFunc` that skips:
- mizu.Ctx creation
- Middleware chain
- Error handler wrapping

```go
func (s *Server) ServeHTTP(w http.ResponseWriter, r *http.Request) {
    // Direct dispatch without mizu overhead
}
```

**3.2 TCP Socket Tuning**

```go
listener, _ := net.Listen("tcp", addr)
tcpListener := listener.(*net.TCPListener)
// Set on accepted connections:
conn.SetNoDelay(true)      // Disable Nagle (important for small responses)
conn.SetReadBuffer(256*1024) // 256KB read buffer
conn.SetWriteBuffer(256*1024) // 256KB write buffer
```

**3.3 Pre-formatted Common Headers**

```go
var commonHeaders = []byte("Accept-Ranges: bytes\r\nServer: LiteIO\r\n")
```

Write headers as a single slice instead of individual `w.Header().Set()` calls.

### Phase 4: Storage Layer Optimizations (Impact: 30% faster)

**4.1 Unified Stat+Open for Reads**

Current flow: `Stat()` → `Open()` → two separate filesystem operations.
Combine into single `Open()` that returns metadata:

```go
// Already done in Open() but handleGetObject calls Stat() FIRST, then Open()
// Fix: Skip the separate Stat() call, get metadata from Open() directly
```

**4.2 O_DIRECT for Large Files on Linux**

Bypass page cache for files >4MB to avoid cache pollution:

```go
//go:build linux
f, _ := unix.Open(path, unix.O_RDONLY|unix.O_DIRECT, 0)
```

Requires aligned buffers (4KB boundary).

**4.3 Pre-allocated File Descriptors**

For write workloads, pre-open the bucket directory fd and keep it cached:

```go
type bucket struct {
    dirFD int // Pre-opened directory file descriptor
}
```

Use `openat()` relative to dirFD instead of full path resolution.

**4.4 Batch Unlink for Delete**

Already implemented (`batchDeleteFiles`). Enhance:
- Increase batch size from 100 to 500
- Use `renameat2` with `RENAME_WHITEOUT` on Linux 5.0+ for instant "delete"

**4.5 In-Kernel Copy for CopyObject**

Already using `copy_file_range` on Linux. Ensure it's used everywhere.

### Phase 5: Response Optimization (Impact: 15% faster)

**5.1 Hand-Written XML for All Responses**

Replace ALL `encoding/xml` usage with hand-written serialization:

```go
func writeListObjectsResultFast(c *mizu.Ctx, result *ListBucketResultV2) error {
    buf := getXMLBuffer()
    defer putXMLBuffer(buf)
    buf.WriteString(`<?xml version="1.0" encoding="UTF-8"?>`)
    buf.WriteString(`<ListBucketResult xmlns="...">`)
    // ... hand-written XML
}
```

This is 5-10x faster than `encoding/xml` (no reflection, no interface dispatch).

**5.2 Inline Small Object Responses**

For reads of cached objects, write the entire response (headers + body) in a single `writev()` call:

```go
// Combine headers and body into single write
headerBuf := formatResponseHeaders(...)
iov := net.Buffers{headerBuf, objectData}
iov.WriteTo(conn)
```

**5.3 Skip ETag for Local Driver**

ETags require hashing the content. For local driver, use file mtime+size as a fast pseudo-ETag:

```go
func fastETag(modTime time.Time, size int64) string {
    return fmt.Sprintf("\"%x-%x\"", modTime.UnixNano(), size)
}
```

### Phase 6: Concurrency Optimizations (Impact: 10% faster)

**6.1 GOMAXPROCS Tuning**

Set `GOMAXPROCS` to match CPU count and ensure goroutine scheduling is optimal.

**6.2 Reduce Goroutine Creation**

Use worker pool for request handling instead of per-request goroutines:

```go
// Not applicable for net/http (already goroutine-per-connection)
// But applicable for storage operations
```

**6.3 Lock-Free Directory Cache**

Replace `sync.RWMutex` with `sync.Map` for the optimized directory cache:

```go

    shards [numDirCacheShards]sync.Map
}
```

Already partially implemented. Ensure ALL paths use the lock-free version.
